Syzygy is a 2d space defender-like. I had wanted to develop a more sophisticated / broad game, and make more of an effort to tailor gameplay and graphics. The basic idea is that a ship is travelling through space and is being attacked by hostile forces, and the ship can be defended using the recharging shield, or enemies defeated by shooting with the point and click turret on top. This was the first project that I worked with a friend on. He did a lot of the graphics, and a small portion of code to do with the enemy behaviour.

Main Menu with nice star field in background

Hangar menu in which the player can buy upgraded systems or weapons

Having started with a small amount of art, and then jumping straight into the gameplay, when it came to implementing menus and different game states, it proved very difficult to effectively switch modes. This proved the point that you should plan projects in advance, and that you should start off with some infrastructure. I essentially restarted the project as a smoother object orientated menu system, and then copied over the code with minimal changes to quickly integrate gameplay. This ended up with a much more robust game-state system and allowed for more flexibility for when I wanted to develop the game further (for example with the hangar, or the possibility of different modes). I originally had a fixed journey, that contained a set number of enemies that had to be killed before the player was destroyed, as well as the endless mode, however it became apparent that this was not well balanced, and a whole set of different (and painstakingly balanced) journeys would be needed to make this system rewarding and last a decent amount of time. I created an endless mode, that remains in the game up to now, that should gradually ramp up in difficulty. The difficulty is fairly arbitrary but has a cap where I would always lose a bit of health every now and again and would ultimately die, when I was testing. As the game was never intended to be widely released, I did not stress too much on the balancing, as for me the joy was in developing the algorithms rather than playing the game.

This project did not have so many hard algorithms as the domino TD, with only a little trigonometry to work out whether a bullet had hit the main or recharging shield probably being the most complicated part. Development of a better UI system and game-state system was helpful as it exposed some of the many ways in which such a system can prove disastrous. I was very pleased with this project, as it had very nice graphics compared to previous projects of mine, it had gameplay that was fun for at least one playthrough, with working upgrades and weapons. I was also happy with the enemy spawning algorithm, and enemy movement (coded by friend), that made the game feel a lot less static.
