In september of 2017 I became very interested in fractals during a computing lesson where I came across 'The Nature of Code' by Daniel Shiffman. I then built a mandelbrot generator using c# in Visual Studio at school. The generator allowed you to set the coordinates and an arbitrary 'zoom' to explore the fractal, but it was still very hard to navigate. The colouring was hard coded, and the palette was not looped, so if you zoomed in further, there was little or no contrast. All in all there were a number of things I wanted to change about this, especially the amount of time it took to draw the fractal, but first I copied the project and edited it to generate the Julia set.

Mandelbrot Generator

With different colour palette

I decided to use processing to build the next mandelbrot generator as it is more suited to programs like this than visual studio. I also tried to make it easier to navigate, by using the mouse rather than input boxes to control the position and zoom. I increased the number of iterations as the user zoomed in to allow more detail, but to keep it running fast when extra detail is not necessary. Additionally, I used a square-root mapped greyscale colour scheme, that is easy to implement but also looks better than a linear one as the larger areas at the outside can be more easily distinguished from each other, and the narrower bands towards the edges of the set achieve a nice smooth gradient.

Self similarity in the Mandelbrot Set

One of the things that fascinates me so much about fractals like the mandelbrot set is that there is self similarity buried in the most obscure places - a mandelbrot shape can be found all over the set and in different rotations, surrounded by different features. see image above. Another interesting feature is the differences in the pattern depending on your escape check. The most common is a^2 + b^2 > 4 (if magnitude of z > 2 then it will escape), however other checks can be used to change the patterns.

(a^2 + b^2) < 4 escape algorithm gives different patterns

(a^2 * b^2) < 4 escape algorithm gives different patterns

Being able to zoom and move naturally was a huge improvement on the visual studio version, as it was much more responsive and easy to use than plugging numbers in. I took a very low level approach to optimisation - the maximum iterations starts low and increases as you zoom. This works quite nicely, but in some areas you do not need so many iterations even if you are zoomed in as far, so this would be an interesting area to optimise.
